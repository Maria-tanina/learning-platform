Класи, структури та об'єкти

Область застосування (контекст) змінної

Кожна змінна доступна у межах певного контексту чи області видимість. Поза цим контекстом змінної вже немає.

Існують різні контексти:

  • Контекст класу. Змінні, визначені лише на рівні класу, доступні у будь-якому методі цього класу. Їх ще називають глобальними змінними чи полями
  • Контекст методу. Змінні, визначені лише на рівні методу, є локальними і доступні лише рамках даного методу. В інших методах вони недоступні
  • Контекст блок коду. Змінні, визначені на рівні блоку коду, також локальні і доступні тільки в рамках даного блоку. Поза власним блоком коду вони не доступні.

    Person tom = new();
    tom.PrintName();
    tom.PrintSurname();
    
    class Person                            // начало контекста класса
    {
        string type = "Person";             // переменная уровня класса
        public void PrintName()             // начало контекста метода PrintName
        {
            string name = "Tom";            // переменная уровня метода

           {                               // начало контекста блока кода
                string shortName = "Tomas"; // переменная уровня блока кода
                Console.WriteLine(type);    // в блоке доступна переменная класса
                Console.WriteLine(name);    // в блоке доступна переменная окружающего метода
                Console.WriteLine(shortName);// в блоке доступна переменная этого же блока
            }                               // конец контекста блока кода, переменная shortName уничтожается
    
            Console.WriteLine(type);        // в методе доступна переменная класса
            Console.WriteLine(name);        // в методе доступна переменная этого же метода
            //Console.WriteLine(shortName); //так нельзя, переменная c определена в блоке кода
            //Console.WriteLine(surname);     //так нельзя, переменная surname определена в другом методе
    
        }       // конец контекста метода PrintName, переменная name уничтожается
    
        public void PrintSurname()      // начало контекста метода PrintSurname
        {
            string surname = "Smith";   // переменная уровня метода
    
            Console.WriteLine(type);        // в методе доступна переменная класса
            Console.WriteLine(surname);     // в методе доступна переменная этого же метода 
        }       // конец конекста метода PrintSurname, переменная surname уничтожается
    
    }   // конец контекста класса, переменная type уничтожается

Тут визначено чотири змінних: type, name, shortName і surname. Кожна з них існує у своєму контексті. Змінна type існує в контексті всього класу Person і доступна в будь-якому місці та блоці коду у методах PrintName та PrintSurname.

Змінна name існує тільки в рамках методу PrintName. Так само як і змінна surname існує у межах методу PrintSurname. У методі PrintName ми можемо звернутися до змінної surname, оскільки він у іншому контексті.

Змінна shortName існує тільки в блоці коду, межами якого є відкриваюча та закриваюча фігурні дужки. Поза його межами змінна shortName не існує і до неї не можна звернутися.

Нерідко межі різних контекстів можна асоціювати з фігурними дужками, що відкриваються і закриваються, як у даному випадку, які задають межі блоку коду, методу, класу.

Працюючи зі змінними треба враховувати, що локальні змінні, визначені у методі чи блоці коду, приховують змінні рівня класу, якщо їх імена збігаються:

Практичні завдання

Питання 1

Що виведе наступний код:

    string name = "Tom";
    Console.WriteLine(Name);
Відповідь

Ця програма не скомпілюється. Метод Console.WriteLine() намагається вивести на консоль значення змінної Name, однак у програмі немає такої змінної. У програмі є лише змінна name. Але оскільки C# реєстрозалежна мова, то name не еквівалентно Name
